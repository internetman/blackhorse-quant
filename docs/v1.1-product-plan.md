# 黑马自选 v1.1 产品方案

## 1. 背景与问题

项目最初以"量化系统"为目标推进，但在实际讨论和开发过程中逐渐暴露出几个核心问题：

- 当前团队并不具备完整量化策略、回测、执行、风控的专业积累
- "量化系统"这个目标过大，容易持续投入但迟迟无法形成可日常使用的产品
- 现阶段更真实的需求，不是自动交易，而是围绕自选股做更快、更准、更有记录的辅助判断

因此，`v1.1` 不再继续沿着"模拟量化交易系统"方向扩展，而是重构为一个面向小圈子使用的 **AI 自选股辅助决策系统**。

## 2. 产品定位

### 一句话定位

一个私有的股票小社群：圈子里有大哥带着大家选股，每只自选股每天有 AI 买卖建议，所有建议都有复盘验证。

### 核心价值

- 给大哥一个统一的信息出口：选股、建议、买卖提醒，不再散落在群聊和线下
- 给圈子成员一个不漏消息的地方：大哥说买、说卖，立刻看到
- 每只自选股每天有 AI 结构化建议，降低人脑分析的速度和信息不完整问题
- 所有建议都有复盘，持续验证"准不准"
- 积累属于圈子自己的判断记录，而不是一次性的临场观点

### 目标用户

- 有意见领袖（大哥）带队的炒股小圈子
- 跟随者/小白：看大哥建议、看 AI 建议来辅助自己的买卖决策
- 需要"统一信息出口 + 辅助决策"而不是"自动交易"的用户

### 解决的核心痛点

```
现状问题：
┌──────────────────────────────────────────────────────────┐
│  大哥在线下开会说了一只票 ──> 有些人没到场，不知道        │
│  大哥在微信群发了建议     ──> 消息刷过去就找不到了        │
│  大哥已经卖了             ──> 其他人不知道，还在持有      │
│  事后复盘                 ──> 没有统一记录，全凭记忆      │
└──────────────────────────────────────────────────────────┘

黑马自选解决：
┌──────────────────────────────────────────────────────────┐
│  大哥选股 ──> 统一发布到黑马圈，所有人立刻看到            │
│  大哥建议买卖 ──> 有时间戳、有理由、有记录                │
│  大哥说卖 ──> 圈子里所有人同步收到                        │
│  每条建议 ──> 系统自动在 T+1/T+3/T+5 复盘                │
│  AI 补充 ──> 每只自选股每天还有 AI 结构化建议             │
└──────────────────────────────────────────────────────────┘
```

## 3. 产品改名建议

`黑马量化` 不再适合作为当前产品名，因为它会让用户误以为这是自动化交易、量化执行、回测驱动的系统。

### 命名方向

保留"黑马"品牌资产，但去掉"量化"表述，向"自选股 + 决策辅助"靠拢。

### 推荐候选名

1. `黑马自选`
2. `黑马决策助手`
3. `黑马看盘助手`
4. `黑马研判`

### 当前建议

优先建议使用：

- 对内/产品名：`黑马自选`
- 对外解释文案：`面向小圈子自选股管理的 AI 决策辅助系统`

原因：

- `黑马自选` 最贴近产品真实核心
- 名字短，直观，好记
- 后续可以自然延展出圈子选股、每日建议、复盘记录等功能

v1.1 先只支持 1 个圈子（你们这个小团体），后续再支持多圈子。

```

│ 角色         │ 权限                                          │
├─────────────┼────────────────────────────────────────────────┤
│ 管理员       │ 邀请用户、管理成员、设置/取消意见领袖、         │

│ 意见领袖     │ 发布选股建议（带 ★ 标识，排序靠前）、          │

├─────────────┼────────────────────────────────────────────────┤
│ 成员         │ 查看所有选股和建议、发布自己的选股建议          │
```

关键区别：

- 领袖发布的选股建议有醒目的 ★ 标识，在列表中排序靠前
- 成员发布的选股建议正常展示，但没有 ★ 标识
- 所有人都能评论（包括在领袖或成员的选股下方）
- 私有持仓只有自己能看

## 5. v1.1 的核心问题

`v1.1` 聚焦解决两个问题：

1. **大哥的建议有统一出口**：选股、买卖建议不再散落在群聊和线下
2. **每只自选股有 AI 建议 + 复盘**：验证建议到底准不准

这意味着：

- 产品重点不是"像交易系统"
- 产品重点是"圈子信息统一 + 建议可验证"
- 第一版先不追求复杂策略与自动下单
- 第一版必须把"圈子 + 建议 + 复盘"的闭环跑通

### 7.4 建议删除或弱化的旧模块

当前旧系统中偏"量化终端展示感"的内容应删除或弱化：

- 模拟持仓跳动
- 模拟成交日志作为主视图
- 复杂策略参数中心
- "同步至实盘环境"式表述
- 伪实盘、伪控制台、伪策略调度感较强的界面

这些内容容易制造"看起来很强"的错觉，但不直接提升日常决策价值。

## 9. 核心交互：股票卡片

`v1.1` 采用 **卡片式展示**，而不是大表格。

原因：

- 自选股天然就是对象集合，卡片更适合承载一只股票的完整判断
- 卡片更适合展示"结论、理由、风险、复盘"
- 比表格更适合小圈子日常使用场景
- 后续更方便做移动端适配

### 9.1 卡片：收起态（轻卡片）

## 13. 隐私方案：公共建议 + 私有持仓

### 13.1 原则

- 股票是公共对象
- 股票建议是公共内容
- 个人账户信息是私有数据
- 个人执行行为不默认公开

### 13.2 数据可见性矩阵

```
┌─────────────────────┬──────────┬──────────┐
│ 数据                 │ 圈子可见  │ 仅自己   │
├─────────────────────┼──────────┼──────────┤
│ 圈子选股             │    ✅    │          │
│ 每日建议             │    ✅    │          │
│ 风险提示             │    ✅    │          │
│ 公共复盘结果         │    ✅    │          │
│ 谁加入了这只股票     │    ✅    │          │
│ 关注原因             │    ✅    │          │
├─────────────────────┼──────────┼──────────┤
│ 是否持有             │          │    ✅    │
│ 成本价               │          │    ✅    │
│ 仓位等级             │          │    ✅    │
│ 个人备注             │          │    ✅    │
│ 个人执行记录         │          │    ✅    │
│ 个人复盘             │          │    ✅    │
├─────────────────────┼──────────┼──────────┤
│ 持股数量             │    不记录           │
│ 总账户资产           │    不记录           │
│ 个人收益排名         │    不做             │
└─────────────────────┴─────────────────────┘
```

### 13.3 为什么不要求公开股数和总资金

因为这会直接带来：

- 隐私压力
- 面子压力
- 使用门槛升高
- 圈子内不必要的比较

## 14. 每日建议的输出格式

为了避免 AI 输出过于发散，建议使用结构化字段。

### 14.1 每日建议字段定义

```
┌─────────────────────┬──────────┬────────────────────────────────────┐
│ 字段                 │ 类型      │ 说明                               │
├─────────────────────┼──────────┼────────────────────────────────────┤
│ id                   │ string   │ 建议唯一 ID                        │
│ symbol               │ string   │ 股票代码 (如 600519.SH)            │
│ name                 │ string   │ 股票名称                            │
│ date                 │ string   │ 建议日期 (如 2026-03-08)            │
│ action               │ enum     │ 可交易/观望/不建议交易/风险升高      │
│ bias                 │ enum     │ 偏低吸/偏突破/偏减仓/暂不参与        │
│ confidence           │ enum     │ 高/中/低                            │
│ pricePlan            │ string   │ 参考价格区间描述                    │
│ summary              │ string   │ 一句话总结                          │
│ reasons              │ string[] │ 核心理由列表                        │
│ risks                │ string[] │ 主要风险列表                        │
│ invalidCondition     │ string   │ 建议失效条件                        │
│ reviewAt             │ string   │ 建议复查时间 (T+1/T+3/T+5)         │
│ generatedAt          │ datetime │ 生成时间                            │
└─────────────────────┴──────────┴────────────────────────────────────┘
```

### 14.2 示例 JSON


## 15. 复盘机制

复盘是产品真正建立价值的核心。

### 15.1 复盘时间线

```
建议日 (T)          T+1           T+3           T+5
   │                 │              │              │
   ▼                 ▼              ▼              ▼
┌──────┐        ┌──────┐      ┌──────┐      ┌──────┐
│ 生成  │        │ 首次  │      │ 中期  │      │ 最终  │
│ 建议  │───────>│ 复盘  │─────>│ 复盘  │─────>│ 复盘  │
│      │        │      │      │      │      │      │
└──────┘        └──────┘      └──────┘      └──────┘
                   │              │              │
                   ▼              ▼              ▼
              记录 T+1        记录 T+3       判定最终
              价格表现        价格表现       有效/失效
```

### 15.2 复盘记录字段

```
┌─────────────────────┬──────────┬─────────────────────────────────┐
│ 字段                 │ 类型      │ 说明                            │
├─────────────────────┼──────────┼─────────────────────────────────┤
│ id                   │ string   │ 复盘记录 ID                     │
│ recommendationId     │ string   │ 关联的建议 ID                   │
│ symbol               │ string   │ 股票代码                        │
│ originalAction       │ enum     │ 当时建议的结论                  │
│ originalSummary      │ string   │ 当时建议的一句话总结            │
│ reviewDate           │ string   │ 复盘日期                        │
│ reviewType           │ enum     │ T+1 / T+3 / T+5                │
│ priceAtRecommend     │ number   │ 建议时的价格                    │
│ priceAtReview        │ number   │ 复盘时的价格                    │
│ priceChange          │ string   │ 涨跌幅 (如 +0.3%)              │
│ verdict              │ enum     │ 有效 / 一般 / 失效              │
│ explanation          │ string   │ 复盘说明                        │
│ generatedAt          │ datetime │ 复盘生成时间                    │
└─────────────────────┴──────────┴─────────────────────────────────┘
```

### 15.3 公共复盘

针对系统当天建议进行复盘，不涉及个人真实交易：

- 当日建议是什么
- T+1 / T+3 / T+5 的价格表现如何
- 如果按建议执行，结果大致如何
- 该建议是有效 / 一般 / 失效
- 失效的主要原因是什么

### 15.4 私有复盘

如果用户录入了自己的私有持仓信息，则可生成只对本人可见的复盘：

- 自己的成本位置
- 当前建议对自己是否适用
- 是否适合继续持有 / 减仓 / 观望
- 如果按建议执行，个人层面的阶段表现如何

## 22. 成功标准

`v1.1` 是否成功，不看"像不像量化系统"，而看下面 5 点：

- 小圈子成员是否愿意每天打开看建议
- 大哥的选股和建议是否被统一记录、不再遗漏
- 建议是否足够结构化、够快读懂
- 复盘是否能持续产出"准不准"的判断
- 用户是否愿意长期把自己的关注股票放进来

如果这 5 点成立，说明产品方向正确。

---

## 23. 最终结论

`v1.1` 应明确转型为：

**一个私有的股票小社群：圈子里有大哥带着大家选股，每只自选股每天有 AI 买卖建议，所有建议都有复盘验证。**

第一版不是量化平台，不是自动交易系统，也不需要假装自己已经具备完整量化能力。

第一版只要做好：

- 黑马圈（圈子 + 角色 + 邀请制）
- 圈子选股（大哥推荐 ★ + 成员关注）
- 每日建议（AI 结构化建议 + 领袖建议排前）
- 结构化卡片（收起态 + 展开态）
- 建议复盘
- 公共建议与私有持仓隔离

它就已经是一个真正能被每天使用的产品。
